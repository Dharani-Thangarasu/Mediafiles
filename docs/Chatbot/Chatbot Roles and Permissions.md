# Chatbot Roles and Permissions

Control who can configure, view, and manage the Document360 Chatbot using role-based access.

---

## Available Roles

| Role            | Permissions                                                      |
|-----------------|------------------------------------------------------------------|
| **Owner**       | Full access: configure, view analytics, manage all settings.     |
| **Admin**       | Configure chatbot settings and view analytics.                   |
| **Editor**      | View analytics only; cannot change chatbot settings.             |
| **Draft Writer**| No chatbot access.                                               |
| **Reviewer**    | No chatbot access.                                               |

---

## Granting Access

1. Go to **Settings → Team & Security → Roles**.
2. Select the role you want to modify.
3. Toggle the relevant **Chatbot** permissions.
4. Save changes.

---

## Chatbot-Specific Permissions

| Permission                    | Description                                           |
|-------------------------------|-------------------------------------------------------|
| **View Analytics**            | Access the chatbot analytics dashboard.               |
| **Manage Chatbot Settings**   | Edit configuration, widget, and escalation settings.  |
| **View Conversation History** | Access stored chat session logs.                      |
| **Export Data**               | Download analytics and conversation data.             |

---

## Best Practices

- Limit **Manage Chatbot Settings** to Admins and Owners.
- Grant Editors read-only access to analytics so they can improve articles based on chatbot data.

---

## Summary

Role-based permissions keep your chatbot secure while allowing the right team members to access the data they need to improve the user experience.
