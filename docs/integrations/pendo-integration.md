# Pendo Integration

Connect Document360 with Pendo to correlate in-app user behavior with knowledge base engagement.

---

## Overview

The Pendo integration allows product teams to track how users interact with documentation alongside in-app behavior, enabling data-driven improvements to both the product and its documentation.

---

## Prerequisites

- A Pendo account with data access
- Document360 project admin access
- Pendo API key or subscription key

---

## Setup Steps

1. Navigate to **Settings → Integrations → Pendo** in Document360.
2. Enter your Pendo **subscription key** or **API key**.
3. Configure the Pendo data source and event mappings.
4. Enable documentation engagement tracking.
5. Set up Pendo guides to surface relevant articles within the product.
6. Click **Save**.

---

## Features

- **Behavioral Analytics** – Correlate product usage data with knowledge base article views.
- **In-App Guides** – Trigger Pendo guides that link to relevant Document360 articles.
- **NPS Correlation** – Identify relationships between documentation engagement and NPS scores.
- **Cohort Analysis** – Analyze documentation usage across different user segments.

---

## Use Cases

- Identify which product features have the highest documentation demand.
- Trigger contextual help guides pointing to knowledge base articles within the product.
- Track whether users who read documentation have better retention rates.

---

## Best Practices

- Define clear event mappings between Pendo actions and documentation views.
- Use Pendo segments to target in-app guides at users most likely to need documentation.
- Review combined analytics monthly to identify content gaps.

---

## Summary

The Pendo integration connects in-app user behavior with documentation engagement, providing product and documentation teams with a unified view of how users learn and succeed.
