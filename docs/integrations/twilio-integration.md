# Twilio Integration

Connect Document360 with Twilio to enable SMS and voice notifications for documentation events and updates.

---

## Overview

The Twilio integration allows teams to send automated SMS messages and voice alerts when key documentation events occur, reaching users through channels beyond email and in-app notifications.

---

## Prerequisites

- A Twilio account with an active phone number
- Document360 project admin access
- Twilio Account SID and Auth Token

---

## Setup Steps

1. Navigate to **Settings → Integrations → Twilio** in Document360.
2. Enter your Twilio **Account SID** and **Auth Token**.
3. Configure the Twilio phone number to use for outgoing messages.
4. Select documentation events that should trigger SMS notifications.
5. Add recipient phone numbers or map them to user profiles.
6. Click **Save**.

---

## Features

- **SMS Alerts** – Send SMS notifications when articles are published or updated.
- **Voice Notifications** – Trigger automated voice calls for critical documentation alerts.
- **Programmable Messaging** – Customize notification message templates.
- **Delivery Tracking** – Monitor message delivery status from the Twilio dashboard.

---

## Use Cases

- Notify on-call engineers via SMS when critical runbooks are updated.
- Send SMS reminders to documentation reviewers with pending review tasks.
- Alert subscribers when high-priority knowledge base articles are published.

---

## Best Practices

- Use Twilio's messaging service for improved deliverability over direct phone numbers.
- Keep SMS notifications concise and include a direct link to the article.
- Test notification flows in Twilio's test environment before going live.

---

## Summary

The Twilio integration extends Document360 notifications to SMS and voice channels, ensuring critical documentation updates reach the right people through the most direct means possible.
