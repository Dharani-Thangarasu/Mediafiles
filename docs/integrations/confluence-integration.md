# Confluence Integration

Migrate and sync content between Confluence and Document360 to modernize your documentation experience.

---

## Overview

The Confluence integration allows teams to import existing Confluence spaces into Document360, keeping documentation accessible and well-organized while leveraging Document360's advanced publishing features.

---

## Prerequisites

- A Confluence Cloud or Confluence Server account
- Document360 project admin access
- Confluence API token

---

## Setup Steps

1. Navigate to **Settings → Integrations → Confluence** in Document360.
2. Enter your Confluence **site URL** and **API token**.
3. Select the Confluence spaces to import or sync.
4. Map Confluence spaces to Document360 categories.
5. Initiate the import or enable continuous sync.
6. Click **Save**.

---

## Features

- **Bulk Import** – Import entire Confluence spaces with page hierarchy preserved.
- **Incremental Sync** – Automatically sync new and updated Confluence pages.
- **Attachment Migration** – Transfer images and file attachments along with content.
- **User Mapping** – Map Confluence authors to Document360 team members.

---

## Use Cases

- Migrate from Confluence to Document360 as a customer-facing knowledge base.
- Keep internal Confluence documentation in sync with external docs.
- Archive legacy Confluence spaces within Document360 for reference.

---

## Best Practices

- Clean up outdated Confluence pages before importing to avoid unnecessary content.
- Review page permissions to control what content becomes publicly accessible.
- Test imports with a small space before migrating large amounts of content.

---

## Summary

The Confluence integration simplifies migrations and enables teams to leverage Document360's publishing capabilities without losing their existing knowledge base investments.
