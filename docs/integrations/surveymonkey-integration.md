# SurveyMonkey Integration

Connect Document360 with SurveyMonkey to gather reader feedback and measure documentation effectiveness through surveys.

---

## Overview

The SurveyMonkey integration allows teams to embed surveys within documentation articles or trigger them at key moments, collecting structured feedback to continuously improve the knowledge base.

---

## Prerequisites

- A SurveyMonkey account with survey creation access
- Document360 project admin access
- SurveyMonkey API credentials

---

## Setup Steps

1. Navigate to **Settings → Integrations → SurveyMonkey** in Document360.
2. Click **Connect with SurveyMonkey**.
3. Authorize Document360 to access your SurveyMonkey account.
4. Select or create surveys to embed in documentation.
5. Configure survey trigger rules (e.g., after reading an article for a set duration).
6. Click **Save**.

---

## Features

- **Survey Embeds** – Embed SurveyMonkey surveys within documentation articles.
- **Exit Surveys** – Trigger surveys when readers leave the knowledge base.
- **NPS Surveys** – Measure documentation satisfaction with Net Promoter Score surveys.
- **Response Analytics** – View aggregated survey results to identify improvement areas.

---

## Use Cases

- Ask readers to rate the helpfulness of troubleshooting articles after reading.
- Run periodic satisfaction surveys to benchmark documentation quality.
- Collect feedback after major documentation updates to validate improvements.

---

## Best Practices

- Keep surveys brief to maximize completion rates.
- Use conditional questions to gather more detailed feedback from dissatisfied readers.
- Act on survey results by creating documentation improvement tasks.

---

## Summary

The SurveyMonkey integration provides a structured feedback mechanism for documentation teams, enabling continuous improvement based on direct reader input.
