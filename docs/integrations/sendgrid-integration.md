# SendGrid Integration

Connect Document360 with SendGrid to manage transactional email notifications for documentation events.

---

## Overview

The SendGrid integration allows teams to send branded, reliable transactional emails when documentation events occur, such as article publications, review requests, and subscriber updates.

---

## Prerequisites

- A SendGrid account with API access
- Document360 project admin access
- SendGrid API key with mail send permissions

---

## Setup Steps

1. Navigate to **Settings → Integrations → SendGrid** in Document360.
2. Enter your SendGrid **API key**.
3. Configure the sender email address and display name.
4. Select the documentation events that trigger email notifications.
5. Customize email templates using SendGrid's dynamic templates.
6. Click **Save**.

---

## Features

- **Transactional Emails** – Send automated emails for article publications and updates.
- **Dynamic Templates** – Use SendGrid's template engine to personalize notification emails.
- **Delivery Analytics** – Track open rates, click rates, and delivery status.
- **Unsubscribe Management** – Automatically handle unsubscribe requests per SendGrid's compliance tools.

---

## Use Cases

- Notify subscribers when new articles are published in their areas of interest.
- Send review request emails to documentation team members via SendGrid.
- Deliver weekly documentation digest emails to knowledge base subscribers.

---

## Best Practices

- Verify your sender domain in SendGrid for improved deliverability.
- Use SendGrid's suppression list management to comply with email regulations.
- Monitor bounce rates and address delivery issues proactively.

---

## Summary

The SendGrid integration powers reliable, branded email notifications for Document360, ensuring that documentation updates reach subscribers and team members effectively.
