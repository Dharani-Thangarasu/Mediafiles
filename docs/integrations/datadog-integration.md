# Datadog Integration

Connect Document360 with Datadog to link monitoring alerts and runbooks, and keep operational documentation aligned with your infrastructure.

---

## Overview

The Datadog integration allows engineering and operations teams to link Datadog monitors and dashboards to Document360 runbooks and incident documentation, enabling faster incident response and operational knowledge management.

---

## Prerequisites

- A Datadog account with API access
- Document360 project admin access
- Datadog API key and application key

---

## Setup Steps

1. Navigate to **Settings → Integrations → Datadog** in Document360.
2. Enter your Datadog **API key** and **application key**.
3. Link Datadog monitors to corresponding Document360 runbooks.
4. Configure alert-triggered documentation notifications.
5. Enable Datadog dashboard embedding in article editor settings.
6. Click **Save**.

---

## Features

- **Runbook Linking** – Attach Document360 runbooks directly to Datadog monitors.
- **Dashboard Embeds** – Embed live Datadog dashboards in documentation articles.
- **Alert Notifications** – Receive Document360 notifications when Datadog alerts fire.
- **Incident Docs** – Auto-create incident documentation when Datadog detects anomalies.

---

## Use Cases

- Link on-call runbooks to Datadog monitors so engineers can act immediately on alerts.
- Embed live Datadog dashboards in infrastructure documentation for real-time visibility.
- Automatically create incident post-mortem documents from Datadog alert events.

---

## Best Practices

- Keep runbooks up-to-date and linked to relevant Datadog monitors.
- Use Datadog tags to categorize monitors and map them to documentation sections.
- Review runbooks after each incident to incorporate lessons learned.

---

## Summary

The Datadog integration connects monitoring and observability with operational documentation, ensuring that on-call teams always have the right runbooks at their fingertips during incidents.
