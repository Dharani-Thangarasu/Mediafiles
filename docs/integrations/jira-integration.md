# Jira Integration

Link Document360 with Jira to connect your documentation process with project management and issue tracking workflows.

---

## Overview

The Jira integration allows teams to reference documentation directly from Jira issues, create documentation tasks automatically, and keep technical content aligned with development sprints.

---

## Prerequisites

- A Jira Cloud or Jira Server account
- Document360 project admin access
- Jira API token

---

## Setup Steps

1. Navigate to **Settings → Integrations → Jira** in Document360.
2. Enter your Jira **site URL** (e.g., `https://yourcompany.atlassian.net`).
3. Provide your Jira **email** and **API token**.
4. Select the Jira project(s) to connect.
5. Configure automation rules as needed.
6. Click **Save**.

---

## Features

- **Issue Linking** – Attach Document360 articles to Jira issues for quick reference.
- **Auto Task Creation** – Automatically create Jira tickets when documentation needs updating.
- **Status Sync** – Reflect Jira issue status changes in documentation workflow states.
- **Webhook Triggers** – Trigger doc updates when Jira sprints close or releases are tagged.

---

## Use Cases

- Create a Jira task for every new feature requiring documentation.
- Link release notes in Document360 to the corresponding Jira release version.
- Track documentation tasks in the same sprint as development work.

---

## Best Practices

- Use Jira labels like `needs-docs` to flag issues requiring documentation.
- Assign documentation tasks to a dedicated writer or team in Jira.
- Review the Jira board at the start of each sprint to plan documentation work.

---

## Summary

The Jira integration brings documentation into the heart of your development workflow, ensuring that content creation is tracked and prioritized alongside feature work.
