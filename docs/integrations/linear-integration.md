# Linear Integration

Connect Document360 with Linear to align documentation tasks with engineering issue tracking and release cycles.

---

## Overview

The Linear integration enables engineering and documentation teams to coordinate content creation alongside software development, ensuring that docs are always in sync with product changes.

---

## Prerequisites

- A Linear workspace with team access
- Document360 project admin access
- Linear API key

---

## Setup Steps

1. Navigate to **Settings → Integrations → Linear** in Document360.
2. Enter your Linear **API key**.
3. Select the Linear teams and projects to connect.
4. Configure issue-to-article linking rules.
5. Set up automation triggers for status changes.
6. Click **Save**.

---

## Features

- **Issue Linking** – Attach Document360 articles to Linear issues.
- **Auto Issue Creation** – Create Linear issues automatically when documentation tasks are assigned.
- **Cycle Sync** – Align documentation milestones with Linear development cycles.
- **Status Mirroring** – Keep issue and article statuses in sync automatically.

---

## Use Cases

- Create documentation tasks in Linear alongside feature development issues.
- Track which Linear issues require documentation updates before release.
- Use Linear cycles to plan documentation sprints aligned with engineering sprints.

---

## Best Practices

- Use Linear labels such as `needs-docs` to flag issues requiring documentation.
- Add documentation tasks to the same Linear cycle as the feature they document.
- Review Linear roadmaps regularly to anticipate upcoming documentation needs.

---

## Summary

The Linear integration ensures that documentation keeps pace with product development by embedding content tasks directly into engineering workflows.
